# V6.5 Configuration Guide\n\n## Overview\n\nV6.5 adds dual-protocol identity binding and ATProto repository support to the Mastopod Federation Architecture. This guide covers configuration for the new components.\n\n## Environment Variables\n\n### Core Identity Configuration\n\n```bash\n# Canonical DID method (did:plc or did:web)\nCANONICAL_DID_METHOD=did:plc\n\n# Pod domain (used for WebID and ActivityPub URIs)\nPOD_DOMAIN=example.com\n\n# Pod context ID (for multi-tenancy)\nPOD_CONTEXT_ID=default\n```\n\n### ATProto Configuration\n\n```bash\n# ATProto PDS endpoint\nATPROTO_PDS_ENDPOINT=https://pds.example.com\n\n# PLC directory URL\nPLC_DIRECTORY_URL=https://plc.directory\n\n# PLC directory timeout (milliseconds)\nPLC_TIMEOUT_MS=30000\n\n# PLC max retries\nPLC_MAX_RETRIES=3\n```\n\n### Redis Configuration\n\n```bash\n# Redis URL for delivery state and caching\nREDIS_URL=redis://localhost:6379\n\n# Repository registry Redis prefix\nREPO_REGISTRY_PREFIX=atproto:repo:\n\n# Repository state TTL (seconds, default 30 days)\nREPO_STATE_TTL=2592000\n```\n\n### Identity Binding Configuration\n\n```bash\n# Identity binding repository type (memory, redis, postgres)\nIDENTITY_BINDING_REPO_TYPE=redis\n\n# Identity binding repository connection string\nIDENTITY_BINDING_REPO_URL=redis://localhost:6379/1\n```\n\n### Signing Configuration\n\n```bash\n# ActivityPods signing API endpoint\nACTIVITYPODS_SIGNING_API_URL=https://activitypods.example.com/api/internal/signatures/batch\n\n# ActivityPods token for authentication\nACTIVITYPODS_TOKEN=your-secret-token\n\n# Signing batch size\nSIGNING_BATCH_SIZE=10\n\n# Signing timeout (milliseconds)\nSIGNING_TIMEOUT_MS=5000\n```\n\n## Redis Keys\n\n### Identity Binding Keys\n\n```\natproto:identity:{canonicalAccountId}          # IdentityBinding document\natproto:identity:by-did:{atprotoDid}           # DID → Account ID mapping\natproto:identity:by-handle:{atprotoHandle}     # Handle → Account ID mapping\natproto:identity:by-actor:{actorUri}           # Actor URI → Account ID mapping\natproto:identity:by-webid:{webId}              # WebID → Account ID mapping\n```\n\n### Repository Registry Keys\n\n```\natproto:repo:{did}                             # Repository state document\natproto:repos                                  # Set of all repository DIDs\n```\n\n### Delivery State Keys\n\n```\natproto:delivery:idempotency:{messageId}       # Idempotency tracking\natproto:delivery:rate-limit:{domain}           # Domain rate limiting\natproto:delivery:slots:{domain}                # Domain concurrency slots\natproto:delivery:blocked-domains               # Set of blocked domains\n```\n\n## RedPanda Topics\n\n### V6.5 Event Topics\n\n```\nap.stream1.local-public.v1                    # Local public activities\nap.stream2.remote-public.v1                   # Remote public activities (post-verification)\nap.firehose.v1                                # Combined public stream\nap.outbound.v1                                # Outbound delivery readiness\nap.inbound.v1                                 # Inbound activities (pre-MRF)\nap.mrf.rejected.v1                            # MRF rejection audit trail\nap.tombstones.v1                              # Delete notifications\nap.identity.provisioned.v1                    # Identity provisioning events\nap.identity.updated.v1                        # Identity update events\nap.link.verified.v1                           # Account link verification events\n```\n\n## Docker Compose NULL,\n  web_id TEXT UNIQUE NOT NULL,\n  activity_pub_actor_uri TEXT UNIQUE NOT NULL,\n  atproto_did TEXT UNIQUE,\n  atproto_handle TEXT UNIQUE,\n  atproto_pds_endpoint TEXT,\n  canonical_did_method TEXT,\n  status TEXT DEFAULT 'active',\n  created_at TIMESTAMP DEFAULT NOW(),\n  updated_at TIMESTAMP DEFAULT NOW(),\n  \n  -- AP keys\n  ap_signing_key_id TEXT,\n  ap_signing_key_pem TEXT,\n  \n  -- AT keys\n  at_signing_key_id TEXT,\n  at_signing_key_pem TEXT,\n  at_rotation_key_id TEXT,\n  at_rotation_key_pem TEXT,\n  \n  -- PLC state\n  plc_update_state TEXT,\n  plc_op_cid TEXT,\n  plc_last_submitted_at TIMESTAMP,\n  plc_last_confirmed_at TIMESTAMP,\n  plc_last_error TEXT,\n  \n  -- Account links\n  ap_also_known_as TEXT[] DEFAULT '{}',\n  at_also_known_as TEXT[] DEFAULT '{}',\n  link_verification_status TEXT DEFAULT 'unverified',\n  link_verified_at TIMESTAMP\n);\n\nCREATE INDEX idx_identity_by_did ON identity_bindings(atproto_did);\nCREATE INDEX idx_identity_by_handle ON identity_bindings(atproto_handle);\nCREATE INDEX idx_identity_by_actor ON identity_bindings(activity_pub_actor_uri);\nCREATE INDEX idx_identity_by_webid ON identity_bindings(web_id);\n```\n\n### Repository State Table\n\n```sql\nCREATE TABLE repository_states (\n  did TEXT PRIMARY KEY,\n  root_cid TEXT NOT NULL,\n  rev TEXT NOT NULL,\n  total_records INTEGER DEFAULT 0,\n  size_bytes BIGINT DEFAULT 0,\n  last_commit_at TIMESTAMP,\n  snapshot_at TIMESTAMP DEFAULT NOW(),\n  commits JSONB DEFAULT '[]',\n  collections JSONB DEFAULT '[]'\n);\n\nCREATE INDEX idx_repo_by_did ON repository_states(did);\n```\n\n## Initialization Steps\n\n### 1. Create Redis Keys\n\n```bash\n# Initialize delivery state keys\nredis-cli SET atproto:delivery:blocked-domains \"{}\"\n```\n\n### 2. Create RedPanda Topics\n\n```bash\nrpk topic create ap.stream1.local-public.v1\nrpk topic create ap.stream2.remote-public.v1\nrpk topic create ap.firehose.v1\nrpk topic create ap.outbound.v1\nrpk topic create ap.inbound.v1\nrpk topic create ap.mrf.rejected.v1\nrpk topic create ap.tombstones.v1\nrpk topic create ap.identity.provisioned.v1\nrpk topic create ap.identity.updated.v1\nrpk topic create ap.link.verified.v1\n```\n\n### 3. Initialize Identity Binding Repository\n\n```bash\n# Create database (if using PostgreSQL)\npsql -c \"CREATE DATABASE mastopod_identity\"\npsql mastopod_identity < identity-schema.sql\n```\n\n### 4. Initialize Repository Registry\n\n```bash\n# Create Redis database for repository state\nredis-cli -n 1 FLUSHDB\n```\n\n## Monitoring\n\n### Health Checks\n\n```bash\n# Sidecar health\ncurl http://localhost:8080/health\n\n# Redis health\nredis-cli ping\n\n# RedPanda health\nrpk cluster health\n```\n\n### Metrics\n\n```bash\n# Repository count\nredis-cli SCARD atproto:repos\n\n# Identity binding count\nredis-cli DBSIZE\n\n# Topic lag\nrpk group list\nrpk group describe {group-name}\n```\n\n## Troubleshooting\n\n### Identity Binding Issues\n\n```bash\n# Check identity binding\nredis-cli GET \"atproto:identity:{canonicalAccountId}\"\n\n# Check DID mapping\nredis-cli GET \"atproto:identity:by-did:{did}\"\n\n# Check handle mapping\nredis-cli GET \"atproto:identity:by-handle:{handle}\"\n```\n\n### Repository State Issues\n\n```bash\n# Check repository state\nredis-cli GET \"atproto:repo:{did}\"\n\n# List all repositories\nredis-cli SMEMBERS atproto:repos\n\n# Check repository count\nredis-cli SCARD atproto:repos\n```\n\n### Delivery State Issues\n\n```bash\n# Check idempotency\nredis-cli GET \"atproto:delivery:idempotency:{messageId}\"\n\n# Check rate limit\nredis-cli GET \"atproto:delivery:rate-limit:{domain}\"\n\n# Check domain slots\nredis-cli LLEN \"atproto:delivery:slots:{domain}\"\n\n# Check blocked domains\nredis-cli SMEMBERS atproto:delivery:blocked-domains\n```\n\n## Performance Tuning\n\n### Redis Configuration\n\n```bash\n# Increase max memory\nmaxmemory 4gb\nmaxmemory-policy allkeys-lru\n\n# Enable AOF persistence\nappendonly yes\nappendfsync everysec\n```\n\n### RedPanda Configuration\n\n```bash\n# Increase partition count for throughput\nrpk topic alter ap.outbound.v1 --partitions 12\n\n# Increase replication factor for durability\nrpk topic alter ap.outbound.v1 --replicas 3\n```\n\n## Security Considerations\n\n1. **Redis**: Use Redis with authentication and TLS\n2. **RedPanda**: Enable SASL authentication and TLS\n3. **Sidecar**: Use environment variables for secrets, never commit tokens\n4. **Keys**: Rotate signing keys regularly\n5. **DID Documents**: Validate all DID documents before trusting\n6. **Account Links**: Reverify links periodically (every 30 days)\n
